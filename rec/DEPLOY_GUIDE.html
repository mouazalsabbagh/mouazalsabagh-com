Deployment Guide

Recommendation Letter System — HostGator (cPanel) Setup
Mouaz AlSabbagh · mo3az.e.s@gmail.com · June 2025

PACKAGE FILES

rec-system/
├── index.html ← The recommendation form (main page)
├── ai.php ← AI proxy: holds Anthropic API key server-side ← NEW
├── collect.php ← DB backend: saves submissions + admin dashboard
└── DEPLOY_GUIDE.html ← This file (read-only reference)
⚠ Why ai.php? Browsers block direct calls to api.anthropic.com (CORS). The proxy runs on your server, holds your API key hidden, and returns results safely to the browser.
Upload index.html, ai.php, and collect.php to your server. This guide stays local.

STEP-BY-STEP DEPLOYMENT

0
Get Your Anthropic API Key (One-time)

The form uses Claude AI to generate letters. You need an API key from Anthropic.

  • Go to console.anthropic.com → sign in or create a free account
  • Navigate to Settings → API Keys → click Create Key
  • Copy the key — it starts with sk-ant-api03-...
  • Open ai.php in a text editor → find line 10 → replace sk-ant-PASTE_YOUR_KEY_HERE with your real key
define('ANTHROPIC_API_KEY', 'sk-ant-api03-xxxxxxxxxxxxxxxxxxxx'); // ← paste your key here
Never share ai.php publicly or commit it to GitHub with your real key inside. The file sits safely on your server — visitors cannot read its PHP source.
Cost: Anthropic charges per token. A recommendation letter costs roughly $0.003–$0.005 USD per generation (fraction of a cent). New accounts get free credits.
1
Create a MySQL Database in cPanel

Log in to your HostGator cPanel → scroll to Databases → click MySQL Databases.

  • Under "Create New Database" → type mouaz_recommendations → click Create Database
  • Under "MySQL Users" → Create a new user, e.g. mouaz_recuser, set a strong password and note it
  • Under "Add User to Database" → select your user + your database → click Add
  • In the privileges popup → check ALL PRIVILEGES → click Make Changes
HostGator prepends your cPanel username to DB/user names. Your actual DB name will be something like username_mouaz_recommendations. Use that exact string in Step 2.
2
Edit collect.php — Update Database Credentials

Open collect.php in a text editor (Notepad++, VS Code, etc.) and update lines 9–13:

define('DB_HOST', 'localhost');           // Always localhost on HostGator
define('DB_NAME', 'cpanelusername_mouaz_recommendations'); // ← your actual DB name
define('DB_USER', 'cpanelusername_mouaz_recuser');          // ← your actual DB user
define('DB_PASS', 'YourStrongPassword');                // ← the password you set
define('ADMIN_PASS', 'YourAdminPass2025!');             // ← change this too
Never upload collect.php with the default CHANGE_THIS_PASSWORD still in it.
3
Choose Where to Host the System

You have two options:

OptionFolder to Upload IntoURL
A — Subfolder (recommended)public_html/rec/yourdomain.com/rec/
B — SubdomainCreate subdomain rec.yourdomain.com → its folderrec.yourdomain.com/

Option A is simpler. Option B looks more professional when sharing the link with recommenders.

To create a subdomain: cPanel → Domains → Subdomains → type rec → Document Root auto-fills → click Create.
4
Upload Files via cPanel File Manager
  • cPanel → File Manager → navigate to your chosen folder (e.g. public_html/rec/)
  • Click Upload → upload index.html and collect.php
  • Verify both files appear in the folder

Alternatively via FTP (FileZilla): Host = ftp.yourdomain.com, Port = 21, use your cPanel username/password.

5
Test the Setup

Visit your URL in a browser — the recommendation form should load:

https://yourdomain.com/rec/

Then test the admin dashboard:

https://yourdomain.com/rec/collect.php?action=admin
  • Login with the ADMIN_PASS you set in Step 2
  • It should show "0 submissions" with an empty table (the DB table auto-creates on first visit)
  • Submit a test recommendation via the form — it should appear in the dashboard within seconds
If you see the dashboard with stats boxes and no PHP errors, you're live!
6
Protect collect.php (Optional but Recommended)

Create a file named .htaccess in the same folder with these rules to block direct browser access to the PHP file except via the form and admin:

# Allow API calls from the form (POST to ?action=save)
# The admin page is password-protected via PHP session anyway
# This blocks random bots from probing collect.php directly
<Files "collect.php">
    Order Deny,Allow
    Allow from all
</Files>
The PHP session-based admin login already provides security. The .htaccess step is optional.

SHARING THE FORM LINK

Send this URL to each person you want a recommendation from:

https://yourdomain.com/rec/

They fill in the form, the AI generates the letter, and a copy is saved to your database automatically. You get an email notification each time.

REQUIREMENTS

  • PHP 7.4+ (HostGator shared hosting uses PHP 8.x by default ✓)
  • MySQL 5.7+ or MariaDB 10+ ✓
  • PDO extension enabled (default on HostGator ✓)
  • No external packages or Composer needed
  • No Node.js required
You're ready to deploy. Two files, one database, five minutes. The table creates itself on first load.
